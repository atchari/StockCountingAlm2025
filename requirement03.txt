เดี๋ยวเรามาเริ่มทำ  dashboard กัน อยากให้ทำตัวเป็นผู้เชี่ยวชาญการวิเคราะห์ข้อมูลหน่อย

ีuse case ที่อยากได้
- เข้ามาหน้า dashboard แล้วเห็นภาพรวมเลย ว่าการรับดำเนินการไปกี่ % แล้ว
- ด้านบนๆ ให้เห็นภาพรวมทั้งบริษัทก่อน และเห็น ภาพรวมระดับคลัง
- แต่สามารถ กดเข้าไปดูในคลังที่มี location ย่อยๆได้
- ntf_FreezeData คือ data ตั้งต้นก่อนนับ , ntf_Counting คือบันทึกการนับ
- วิธีเทียบ ให้ดู query นี้เป็นตัวอย่าง

select f0.qty as freezeQty,C0.qty as countQty,f0.* from ntf_FreezeData f0
left join ntf_Counting C0 on F0.whsId = C0.whsId 
and  ISNULL(F0.binId, 0) = ISNULL(C0.binId, 0)
and f0.sku = C0.sku
and isnull(f0.batchNo, '') = isnull(C0.batchNo, '')
where f0.whsId = 2

- % การนับ ให้ดูแถวที่มีการนับแล้ว เทียบกับจำนวนแถวของ FreezeData ไม่เกี่ยวว่าที่นับมาจำนวนต้องตรงกับที่ freeze ไว้
- คลังไหน/location ไหนไม่มีข้อมูล freeze data ให้ แสดงว่า ยังไม่มีข้อมูลตั้งต้น
- ถ้า location ไหน นับครบแถวแล้ว ให้แสดงว่านับครบ ถ้าเริ่มมีใส่ข้อมูลแล้ว แสดงว่ากำลังนับ
- มีอะไรบอก ว่าตรงนี้มียอดไม่ตรงกับ freeze นะ แต่ไม่เกี่ยวกับ % การนับ เพื่อให้ auditor อาจเข้าไปดูตรวจสอบแก้ไขตัวเลข

และ เสนอแนะแล้วทำออกมาหน่อย ว่าหน้า dashboard ควรมีอะไรในการนับ stock แล้วทำออกมาเลย หน้านี้ให้ผู้บริหารดู
